ASE Unit 0 • Lesson 0.2

Engineering Notebook & Documentation Refresher

Refresh the documentation routines used for sketches, calculations, CAD evidence, test data, design decisions, and reflections.

Back to Unit 0 Previous: 0.1 Next: 0.3
Unit Connection

Mission Readiness: Safety, Tools & Engineering Systems

Focus Question: How do engineers document work so another person can understand, trust, and build from it?

✓Lesson Snapshot

Student Objective

I can set up a ASE notebook system for calculations, sketches, test data, CAD evidence, and design decisions.

Main Activity

Notebook setup, sample evidence review, documentation quality check, and first ASE notebook entry.

Deliverable

Engineering notebook setup entry

Tools / Materials

Engineering notebook, graph paper, course resource library

1ProblemUnderstand the challenge and why it matters.

Engineering work loses value when it cannot be understood later. A notebook is not just a place to store papers; it is evidence of your design process, calculations, tests, and decisions.

2ConceptLearn the engineering idea or skill.

Strong ASE documentation should be dated, labeled, complete, readable, and connected to a decision. It should include sketches, calculations, measurements, CAD screenshots, photos, test data, and short explanations when needed.

3ApplyUse the skill in a guided task.

Review examples of weak and strong engineering documentation. Identify what makes the strong examples easier to follow, then set up your ASE notebook expectations for this course.

4DocumentRecord your evidence and decisions.

Create a notebook entry titled “ASE Documentation System.” Include your table-of-contents setup, file naming expectations, and three rules for making notebook evidence useful.

5ReviewCheck quality and identify your next step.

Before moving on, check that your notebook system can support future VEX builds, structure testing, rover programming, Fusion work, and final project evidence.
